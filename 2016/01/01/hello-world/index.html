第一篇 谈谈使用的软件

2016-01-01

首先很高兴有人能看到我的博客Octopus! 这里有我的github github 需要的话都拿走好了。

浏览器

Chrome

  • 快，谷歌最拿的出手的地方，v8引擎想当年就是凭借这一点跟微软抢到的市场，直至今日，各浏览器的极速版都是使用的谷歌内核。搜索引擎方面，虽然Google被墙的厉害，不过凭借各种翻墙方法还是能体会到，谷歌搜索的速度。
  • 简洁的界面，并没有繁杂的广告，标签栏地址栏基本上什么都没有，不管是不是抄袭还是模仿，总之谷歌的UI做的我很喜欢。
  • 拓展以及同步，谷歌的拓展还是很庞大的，可以通过谷歌商店还有网上的插件安装各种拓展。账号同步使得不会因为重做系统或者换电脑丢失自己网站收藏。
  • H5支持率高。

火狐

  • 源代码全部开放，使得各种技术大牛自己写插件，完全自定义，当然菜鸟什么的也可以找到相当多的插件拓展（也有很多恶意插件）。
  • 开发工具强大，也是因为插件多的原因，各种开发者插件层出不穷。
  • 安全，公认的最安全的浏览器，漏洞修补速度相当快。小，稳定性很好。
  • H5支持率高

IeTester

  • 一提到IE，我就根本想不到优点。由于当年微软的捆绑销售，使得IE6占据庞大的市场，直至今日，还有大量用户使用者XP系统。所以就必须考虑IE的兼容问题。
  • 还有频繁更新的版本，保留原版的作法现在看来真的很糟糕。各个版本之间的差异问题，使前端需要做很多hack操作，增加了很多没有意义的代码。
  • 对H5的兼容问题，IE10以后的版本对H5的兼容性还算良好，也就只能算是良好，普通的效果都可以实现，而9之前的效果，简直是惨不忍睹。
  • IEtester是一个多版本IE应用，用来测试网站在不同IE版本中的兼容性。

编辑器

Dreamweaver

  • 快速的制作速度，DW将Firework以及PhotoShop集成进来，使得我们可以快速的使用并且更改设计图。
  • 直观的显示效果，可以从设计窗口更改网站界面，精准定位，在设计视图点击某一元素，则会直接跳到代码处。
  • DW更适合偏向设计方面，对代码要求不高的人员，因为超强的设计界面以及编辑界面都并不会让你的代码更简洁。
  • DW的界面显示并不能完全等同于浏览器，而且会出现很多兼容问题。软件比较大，而且经常出现卡死的现象。设计窗口生成代码，并不简洁，更谈不上优美。

Sublime

  • 作为一个越来越受欢迎的文本编辑器，Sublime有很多别的IDE不具备却很让人喜欢的优点。比如说多重修改，Sublime支持多光标修改，如果修改十个LI中的href，只要建立十个光标就能同时修改，简单快速。
  • 快速准确的代码生成，新建文件只要简单的几个快捷键,通过各种快捷键的使用使得我们建站速度大大增快，也减少了我们敲重复代码的次数。
  • 布局合理，界面美观，代码块视口可以直观的看到全局的代码查找或是切换都很快速。
  • 插件齐全，只要想的到就能找得到，安装也很方便。
  • 快！真的很快，打开运行，使用都非常快，并没有DW或者WS打开时加载项目的停滞。
  • 只是每次使用经常弹出来的付费窗口实在让人反感，还有对项目的管理并不是很好，毕竟只是一个文本编辑器，项目管理方面弱点严重。

WebStrom

  • WebStrom被誉为前端开发神器，那么自然就有其过人之处。首先加入图片时会显示图片高宽，做网页的都应该有体会，切文件找图片宽高还是很麻烦的。
  • 重构重命名功能，更换标签名字方便简单。更改文件名称，会自动修改所有引用，使得使用起来非常的舒服。
  • 不过集成了这么庞大的功能，整体体积自然变得很大，运行内存大，启动时间长真的是它的硬伤，有这样一句话，你如果用WebStrom上班时候打开，下班时候关闭就好了，因为启动实在太慢。

Hbuider

  • HBuilder是DCloud（数字天堂）推出的一款支持HTML5的Web开发IDE。这个是我近期一直在使用的软件，快，是Hbuider的一大特色，无论是代码联想，还是发布启动，都很快。而且自带发布功能，界面美观，免安装，一键生成等等。
  • 正在开发完善的IDE，存在一些bug。

Git

Git Bash

  • 版本管理工具无外乎两种，SVN以及github，做前端的更喜欢github一些，上面开源项目很多。
  • github强大的版本管理功能，使得再也不用担心代码反复修改的问题（还是反感反复修改代码的）。
  • 及时上传下载功能，使得团队开发同一项目时可以分部实施，分支功能也使得每个成员的代码不会由于bug相互干扰。
  • 缺点就是，每次使用都要敲命令行，习惯了感觉还好，刚开始的时候真的很苦恼。

PhotoShop

  • 有人说前端不需要了解掌握PS，很明显这是不对的，如果只依靠美工来切图，对自己的开发过程会带来很大的困扰，每换一个美工就要习惯一个美工的切图习惯，这样是很苦恼的问题，所以前端还是需要掌握PS基础的。我本是设计入门的，虽说不做设计很多年，不过切切图，改改样式还是可以胜任的。
