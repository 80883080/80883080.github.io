第一篇 谈谈使用的软件

2016-01-05

首先很高兴有人能看到我的博客Octopus! 这里有我的github github 需要的话都拿走好了.下面谈谈正在使用的软件

浏览器

Chrome

  • 一方面觉得Chrome真的是相当快，还有另一方面就是我们项目正在使用谷歌的内核，所以开发都是使用chrome的，虽然在中国被墙的厉害，不过还是能通过一些方式翻出去的，我是用的谷歌访问助手，还是蛮好用的。 Webkit内核

IeTester

  • 作为前端当然需要配置各种浏览器测兼容哒，针对IE不同版本，通过IETester，测兼容，不过IE7本地版不能测试。 IE内核

火狐

  • 无论前段后台都对火狐的开发者模式钟爱有加 Firefox内核

编辑器

Dreamweaver

  • Dreamweaver是Macromedia开发发布的，后来被Adobe给收购了，它因多屏幕预览，实时视图等优点被很多开发者喜爱。
  • 刚开始学习前端的时候就一直使用 Dreamweaver，虽说可以直观的从视图口看到网站的样子，不过具体使用中并没有几个人去用，而且，DW由于集成过多设计元素，显得很笨重。经常遇到卡死的情况。而且最重要的是每次安装都需要破解一个多G，确实太大了。

Sublime

  • 我刚发现Sublime Text的时候是如获至宝，它有漂亮的用户界面和强大的功能，例如代码缩略图，Python的插件，代码段等。还可自定义键绑定，菜单和工具栏。Sublime Text 的主要功能包括：拼写检查，书签，完整的 Python API ， Goto 功能，即时项目切换，多选择，多窗口等等。
  • 不过缺点还是很明显的，很多项目都需要发布localhost查看，在自己配tomcat等等的，显然很麻烦。

WebStrom

  • WebStrom是jetbrains公司旗下一款JavaScript 开发工具。被广大中国JS开发者誉为“Web前端开发神器”、“最强大的HTML5编辑器”、“最智能的JavaScript IDE”等。使用WebStrom可以随时的发布，界面也很美观。解决了上面的问题。
  • 不过用惯了Sublime 还是觉得这种打开要加载很久的软件很笨重。

Hbuider

  • HBuilder是DCloud（数字天堂）推出的一款支持HTML5的Web开发IDE。这个是我近期一直在使用的软件，快，是Hbuider的一大特色，无论是代码联想，还是发布启动，都很快。而且自带发布功能，界面美观，免安装，一键生成等等。
  • 正在开发完善的IDE，存在一些bug。

Git

Git Bash

  • 学习使用github的时候就是通过，Git敲入命令执行的，所以一直没有使用桌面版的习惯。

PhotoShop

  • 有人说前端不需要了解掌握PS，很明显这是不对的，如果只依靠美工来切图，对自己的开发过程会带来很大的困扰，每换一个美工就要习惯一个美工的切图习惯，这样是很苦恼的问题，所以前端还是需要掌握PS基础的。我本是设计入门的，虽说不做设计很多年，不过切切图，改改样式还是可以胜任的。
