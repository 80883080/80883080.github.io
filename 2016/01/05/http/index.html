个人总结HTTP

2016-01-05 • 标签: HTTP GET POST 404 200

首先很高兴有人能看到我的博客Octopus! 这里有我的github github 需要的话都拿走好了。
最近遇到几个HTTP方面的问题，其中包括HTTP的一些方法，和HTTP的几种状态，所以总结一下。

HTTP

  • 划楞下脑袋里面的HTTP知识，首先进入脑袋的就是，他是一个传输协议，要通过几次握手，有GET,POST等方法，有五种状态，（说到这里我不得不吐槽一下，做前端实在是太广泛了，前端需要的
    H5+C3需要掌握，各种js框架，UI框架，移动框架需要会，PS需要会，做后台的Node需要会，HTTP协议还要会，大家未免对前端要求也太高了点）。言归正传，我们首先想到的也就只有这几个，反正我
    是。

状态

  • 100+ 传输继续
  • 200+ 传输成功
  • 300+ 重定向 （服务器根据请求可执行多种操作）
  • 400+ 请求出错（服务器端检测到与其配置不符合的请求会直接回复400错误给客户端）
  • 500+ 服务器错误（说明服务器无法解析代码）

方法

  • GET 发送一个请求来获得服务器上的某个资源。资源通过http头和数据的形式返回。方法$.getJSON $.ajax $.get

  • POST 向服务器提交数据。 方法$.post

  • HEAD 发送请求，获取服务器头。

  • PUT HTML不支持。不用了解
  • DELETE 删除资源，不常见，反正我没用过。
  • OPTIONS 获取方法。

跨域

  • 前端经常遇到跨域问题，通常解决方法，用JSONP方式，代理服务器方式，以及H5的postMessage。
  • JSONP应该是我用的最多的方式，由于script可以指定src位置，所以动态的指定src为目标服务器的位置，需要后台配合，通过callback参数。
  • 代理服务器，有一个中间服务器传输数据，在不跨域的位置建立服务器获取数据，然后转发给前端。
  • H5的postMessage 调用postMessage方法的window对象是指要接收消息的那一个window对象，该方法的第一个参数message为要发送的消息，类型只能为字符串；第二个参数targetOrigin用来限定接收消息的那个window对象所在的域，如果不想限定域，可以使用通配符 * 需要接收消息的window对象，可是通过监听自身的message事件来获取传过来的消息，消息内容储存在该事件对象的data属性中。
